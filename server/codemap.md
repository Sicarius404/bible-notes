# server/

## Responsibility
The PocketBase backend for bible-notes. Owns the server runtime configuration, Docker image definition, startup logic, database files, JS hooks, schema migrations, and seed data.

## Files
- **Dockerfile.pocketbase** — Custom production Docker image based on `ghcr.io/muchobien/pocketbase:0.37.2`. Bakes hooks (`./hooks` → `/pb_hooks`), migrations (`./pb_migrations` → `/pb_migrations`), and a custom entrypoint into the image. This avoids bind-mount fsnotify restart loops in production.
- **entrypoint.sh** — Custom startup script that replaces the upstream entrypoint. Replicates superuser auto-creation (`pocketbase superuser upsert`) from env vars, then launches `pocketbase serve` with explicit flags: `--http=0.0.0.0:8090`, `--dir=/pb_data`, `--publicDir=/pb_public`, `--hooksDir=/pb_hooks`, `--migrationsDir=/pb_migrations`.
- **pocketbase** — PocketBase binary (compiled Go executable, used in development).
- **CHANGELOG.md** — Release history for the server.
- **LICENSE.md** — License file.

## Subdirectories
- **hooks/** — JS hook source files (used in Dockerfile to bake into `/pb_hooks`). These are the authoritative hook sources.
- **pb_hooks/** — Duplicate of `hooks/` mounted at `/pb_hooks` at runtime. Identical content to `hooks/` — exists for local development where bind-mount is acceptable.
- **pb_migrations/** — PocketBase schema migration files (JS). Applied in order on server start.
- **pb_data/** — Local PocketBase data directory (SQLite databases + generated type stubs).
- **seed/** — Static seed data assets (reading plan JSON files).

## pb_data/
- **data.db** — Main PocketBase SQLite database containing all collections and records.
- **auxiliary.db** — PocketBase auxiliary SQLite database (internal metadata, realtime state, etc.).
- **types.d.ts** — Auto-generated TypeScript type declarations for the PocketBase JSVM environment (cron, router, hooks APIs, model classes, etc.). Generated by PocketBase, not hand-written.

## Integration
- **Build pipeline**: `Dockerfile.pocketbase` is built by Coolify, producing an image with hooks and migrations baked in.
- **Traefik routing**: PocketBase serves API at `/api/` and admin at `/_/` on port 8090, routed by Traefik rules defined in `docker-compose.production.yml`.
- **Frontend**: The Next.js frontend (apps/web) communicates with PocketBase via the `packages/pocketbase-client` wrapper.
- **Deployment**: Managed by Coolify. The `command` key must NOT be overridden in production compose — the entrypoint handles all flags.
